# Epic Create Skill

Create RHOAIENG epics from an approved RHAISTRAT strategy. This skill ONLY creates epics — no child tasks, stories, spikes, or sub-tasks.

## Usage

```
/epic-create RHAISTRAT-1536
/epic-create RHAISTRAT-1536 --headless
/epic-create RHAISTRAT-1536 --dry-run
/epic-create RHAISTRAT-1536 --headless --dry-run
```

| Flag | Behavior |
|------|----------|
| *(no flags)* | Interactive mode. Present the epics, ask for confirmation, then create on approval. |
| `--headless` | Non-interactive. Create all epics automatically. |
| `--dry-run` | Output the epic definitions but do NOT create any tickets in JIRA. |
| `--headless --dry-run` | Non-interactive + no creation. Output the structured epic list silently. |

## Instructions

### Jira API

All Jira operations use the bundled script. Requires `JIRA_SERVER`, `JIRA_USER`, and `JIRA_TOKEN` environment variables.

```bash
# Fetch an issue (returns JSON)
python3 ${CLAUDE_SKILL_DIR}/scripts/jira_ops.py get-issue RHAISTRAT-1536

# Fetch comments for an issue (returns JSON array)

# Create an issue (pass JSON payload as argument, returns created issue JSON)
python3 ${CLAUDE_SKILL_DIR}/scripts/jira_ops.py create-issue '{"fields": {...}}'

# Update an issue
python3 ${CLAUDE_SKILL_DIR}/scripts/jira_ops.py update-issue RHOAIENG-12345 '{"fields": {...}}'

# Search with JQL
python3 ${CLAUDE_SKILL_DIR}/scripts/jira_ops.py search 'project = RHOAIENG AND ...'
```

### Step 0: Precondition Checks

Before doing any work, verify ALL of the following. If any check fails, report the failure and exit.

#### 0a: Check strat review is approved

Read the strat review artifact:
```
/app/artifacts/strat-reviews/{STRAT-KEY}-review.md
```

Check the frontmatter:
- `recommendation` must be `approve`
- If `recommendation` is `revise` or `reject`, exit with: `"Strat review recommendation is '{recommendation}', not 'approve'. Cannot create epics until the strategy is approved."`

#### 0b: Check security review passed

Read the security review artifact:
```
/app/artifacts/security-reviews/{STRAT-KEY}-security-review.md
```

Check the frontmatter:
- `verdict` must be `PASS`
- If `verdict` is `CONCERNS` or `FAIL`, exit with: `"Security review verdict is '{verdict}', not 'PASS'. Cannot create epics until security concerns are resolved."`
- If the file does not exist, exit with: `"No security review found for {STRAT-KEY}. Run strat-security-review first."`

#### 0c: Check for existing epics (idempotency)

```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/jira_ops.py search 'project = RHOAIENG AND issuetype = Epic AND parent = {STRAT-KEY}'
```

If the search returns one or more issues, epics have already been created. **Do not create duplicates.** Output:
`"RHAISTRAT-{key} already has {N} child epic(s) in RHOAIENG: {keys}. Skipping."`

### Step 1: Read the RHAISTRAT Ticket

Fetch the RHAISTRAT ticket and its comments:

```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/jira_ops.py get-issue {STRAT-KEY}
python3 ${CLAUDE_SKILL_DIR}/scripts/jira_ops.py get-comments {STRAT-KEY}
```

Extract:

| Field | Where to Find | Used For |
|-------|--------------|----------|
| Summary | `summary` | Epic titles |
| Description | `description` | Epic scope and acceptance criteria |
| Issue Type | `issuetype` | Feature vs Initiative |
| Components | `components` | RHOAIENG component field |
| Priority | `priority` | Inherited by epics |
| Assignee | `assignee` | Default epic assignee |
| Labels | `labels` | Check for release candidate labels |
| Fix Version | `fixVersions` | Inherited by epics |
| Due Date | `duedate` | Inherited by integration epics |

Also check for:
- **Strategy section**: Look for `h2. Strategy (AI Generated by Agentic SDLC Pipeline)` — contains Affected Components, Impacted Teams, High Level Requirements, Dependencies, Risks, and Effort Estimate.

### Step 2: Determine Which Epics to Create

Only create **epics** — no child tasks, stories, or spikes. Those will be created separately later.

```
Always create:
  [Eng] Epic — engineering implementation work
  [QE] Epic — test validation work

If new component being added to RHOAI:
  Integration epic — based on target maturity:
    Dev Preview  → Reference RHOAIENG-31244 template
    Tech Preview → Reference RHOAIENG-31290 template
    GA           → Reference RHOAIENG-31303 template
                   + Stakeholder Sign-Off Epic

If maturity level is unknown, ask the user (interactive) or skip integration epics (headless).
```

**Summary of possible epics:**

| Epic | When to Create |
|------|---------------|
| `[Eng RHAISTRAT-XXXX] {title}` | Always |
| `[QE RHAISTRAT-XXXX] {title} validation` | Always |
| `Dev Preview - ODH and RHOAI Integrations [{Component}/{Feature}]` | New component targeting DP |
| `Tech Preview - ODH and RHOAI Integrations [{Component}/{Feature}]` | New component targeting TP |
| `General Availability - ODH and RHOAI Integrations [{Component}/{Feature}]` | Component graduating to GA |
| `[RHAISTRAT-XXXX] Stakeholder Sign Offs` | GA features |

### Step 3: Generate Epic Definitions

#### [Eng] Epic

**Title**: `[Eng RHAISTRAT-{key}] {concise feature title}`

**Description** (Jira markup):
```
h2. Overview

Implement {one-sentence summary}, as specified in [RHAISTRAT-{key}|https://redhat.atlassian.net/browse/RHAISTRAT-{key}].

h2. Scope

* {scope item 1 — derived from RHAISTRAT requirements/acceptance criteria}
* {scope item 2}
* ...

h2. Acceptance Criteria

* {criterion 1 — mapped from RHAISTRAT, rewritten for engineering}
* {criterion 2}
* ...
```

#### [QE] Epic

**Title**: `[QE RHAISTRAT-{key}] {concise description} validation`

**Description** (Jira markup):
```
QE epic for RHAISTRAT-{key} -- {feature title}.

h2. Scope

h3. {test area 1}
* {test scenario}
* {test scenario}

h2. Dependencies

* {dependency 1}
```

#### Integration Epic (if applicable)

If the template exists in this JIRA instance, instruct the user to clone it:
```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/jira_ops.py get-issue RHOAIENG-{template_key}
```

If the template does not exist, skip the integration epic and note:
> **Integration epic skipped**: The {DP/TP/GA} integration template (RHOAIENG-{key}) does not exist in this JIRA instance.

#### Stakeholder Sign-Off Epic (GA only)

**Title**: `[RHAISTRAT-{key}] Stakeholder Sign Offs`

This is an empty epic — the 8 sign-off tasks will be added separately.

### Step 5: Create or Output

#### Interactive mode (no flags)
1. Present epics (Step 4 format).
2. Ask: "Should I create these epics in JIRA?"
3. On approval, create epics.

#### `--headless` mode
1. Present epics, then create them immediately.

#### `--dry-run` mode
1. Present epics with full field definitions.
2. End with: `**Dry run complete. {N} epics would be created.**`

#### Creation procedure

Use `python3 ${CLAUDE_SKILL_DIR}/scripts/jira_ops.py create-issue '<JSON>'` to create each epic, setting:
- `project`: `{"key": "RHOAIENG"}`
- `issuetype`: `{"id": "10000"}` (Epic)
- `parent`: `{"key": "RHAISTRAT-{key}"}`
- `summary`: Epic title
- `description`: Generated description
- `priority`: Inherited from RHAISTRAT
- `components`: Inherited from RHAISTRAT
- `fixVersions`: Inherited from RHAISTRAT
- `customfield_10011`: Epic Name (same as summary)
- `customfield_10855`: Target Version (mirror of fixVersions)

After creation, output a summary table AND write a receipt artifact to
`/app/artifacts/epic-receipts/{STRAT-KEY}.md`:

```markdown
---
strat_key: RHAISTRAT-{key}
created_at: "YYYY-MM-DDTHH:MM:SSZ"
epics:
  - key: RHOAIENG-XXXXX
    title: "[Eng RHAISTRAT-{key}] ..."
    type: eng
  - key: RHOAIENG-XXXXX
    title: "[QE RHAISTRAT-{key}] ..."
    type: qe
---
```

Create the `/app/artifacts/epic-receipts/` directory if it does not exist.
This receipt file is used by the workflow engine to skip re-runs.

Also output the summary table to stdout:
```
## Created Epics

| Key | Title | Parent |
|-----|-------|--------|
| RHOAIENG-XXXXX | [Eng RHAISTRAT-{key}] ... | RHAISTRAT-{key} |
| RHOAIENG-XXXXX | [QE RHAISTRAT-{key}] ... | RHAISTRAT-{key} |
```

In `--dry-run` mode, do NOT write the receipt artifact.

|------|-----|

```
RHAISTRAT Feature (hierarchyLevel: 2)
  └── RHOAIENG Epic (hierarchyLevel: 1) — linked via parent field
```

## Guidelines

- **Only create epics.** Do not create tasks, stories, spikes, or sub-tasks. Those are handled by separate skills.
- **Do not create integration epic tasks manually.** If the template exists, instruct the user to clone it. If it doesn't exist, skip it entirely.
- **The RHAISTRAT ticket is the parent of all RHOAIENG epics.** Use the `parent` field.
- **Prefix conventions**: `[Eng RHAISTRAT-XXXX]` for engineering, `[QE RHAISTRAT-XXXX]` for validation.
